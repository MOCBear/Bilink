# Bilink 个人交互主页系统

## 1. 项目概述

### 1.1 项目简介

Bilink 是一个采用扁平现代化设计风格的个人交互主页系统，集成了类似 iOS 26 系统的液态玻璃（Glassmorphism）视觉效果，为用户提供一个美观、交互友好的个人展示平台。

### 1.2 项目目的

- 提供一个现代化、视觉吸引力强的个人主页，展示用户的个人信息、技能、项目和联系方式
- 实现完整的用户权限管理，区分普通访客和管理员角色
- 采用前后端分离架构，提高系统的可维护性和扩展性
- 优化用户体验，实现响应式设计和流畅的动画效果

### 1.3 核心价值

- **视觉吸引力**：采用液态玻璃效果和现代设计语言，打造具有层次感和深度的界面
- **功能完整性**：提供个人信息展示、技能管理、项目展示和联系功能
- **管理便捷性**：管理员可以通过控制面板轻松编辑和管理所有内容
- **安全性**：实现完整的登录验证系统，保护管理功能的安全
- **响应式设计**：适配不同设备尺寸，确保在手机、平板和桌面端都有良好的显示效果

## 2. 功能模块说明

### 2.1 前端功能模块

#### 2.1.1 主页模块

**核心功能**：
- **Hero 区域**：展示用户头像、姓名、头衔、简短介绍和社交链接
- **关于我**：详细介绍用户的个人背景和经历
- **技能专长**：展示用户的专业技能和熟练度
- **项目作品**：展示用户的项目案例，包括项目名称、描述、链接和标签
- **联系方式**：提供用户的邮箱和所在地信息
- **隐藏登录触发**：通过点击页面3次触发显示管理员登录按钮

**技术实现**：
- 使用 React 组件化开发
- 集成液态玻璃（Glassmorphism）视觉效果
- 响应式设计，适配不同设备尺寸
- 平滑的动画和过渡效果
- 骨架屏加载，提升用户体验

#### 2.1.2 登录模块

**核心功能**：
- 管理员账户登录
- 登录状态验证
- 错误信息反馈

**技术实现**：
- 使用 React 表单处理
- JWT 令牌认证
- 客户端状态管理

#### 2.1.3 管理员控制面板

**核心功能**：
- **基本信息管理**：编辑用户姓名、头衔、头像、简介和关于我内容
- **技能管理**：添加、编辑、删除技能和熟练度
- **项目管理**：添加、编辑、删除项目信息
- **社交链接管理**：添加、编辑、删除社交平台链接
- **联系方式管理**：编辑邮箱和所在地信息
- **登录状态管理**：退出登录功能

**技术实现**：
- 基于角色的权限控制
- 表单验证和数据提交
- 实时数据更新和反馈

### 2.2 后端功能模块

#### 2.2.1 认证模块

**核心功能**：
- 用户登录验证
- JWT 令牌生成和验证
- 管理员权限检查

**技术实现**：
- 使用 Express.js 路由
- bcrypt 密码加密
- JWT 认证中间件

#### 2.2.2 个人资料管理模块

**核心功能**：
- 获取个人资料数据（公开访问）
- 更新个人资料数据（管理员权限）

**技术实现**：
- RESTful API 设计
- 数据验证和错误处理

#### 2.2.3 数据存储模块

**核心功能**：
- 数据文件的创建和初始化
- 数据的读取和写入
- 默认数据配置

**技术实现**：
- JSON 文件存储
- 文件系统操作
- 数据结构管理

**密码存储位置**：
- 密码存储在 `backend/data/data.json` 文件中
- 密码以加密哈希的形式存储，使用 bcryptjs 库进行加密
- 首次运行时，系统会自动创建默认的管理员账户和加密密码
- 密码哈希值无法直接解密，确保了密码的安全性

### 3.2 架构设计

#### 3.2.1 整体架构

Bilink 系统采用前后端分离的架构设计，具体如下：

```
┌─────────────────────┐       HTTP       ┌─────────────────────┐
│                     │    Request/      │                     │
│    前端应用         │  Response        │    后端服务         │
│  (React + Vite)     │ ──────────────── │  (Express.js)       │
│                     │                  │                     │
└─────────────────────┘                  └─────────────────────┘
                                               │
                                               │
                                               ▼
┌─────────────────────┐                  ┌─────────────────────┐
│                     │                  │                     │
│    浏览器           │                  │    数据存储         │
│  (用户界面)         │                  │  (JSON 文件)        │
│                     │                  │                     │
└─────────────────────┘                  └─────────────────────┘
```

#### 3.2.2 前端架构

前端采用组件化的架构设计，主要包括以下层次：

1. **页面层**：包含主页、登录页和管理员控制面板三个主要页面
2. **组件层**：各个功能模块的组件，如 Hero 组件、技能组件、项目组件等
3. **服务层**：API 调用和数据处理
4. **状态管理**：使用 React Context API 进行全局状态管理，特别是认证状态
5. **样式层**：全局样式和组件样式，实现液态玻璃效果

#### 3.2.3 后端架构

后端采用 MVC（Model-View-Controller）架构的变体，主要包括以下层次：

1. **路由层**：处理 HTTP 请求和响应
2. **控制器层**：实现业务逻辑
3. **中间件层**：处理认证和权限验证
4. **数据层**：管理数据的存储和检索

### 3.3 数据流

#### 3.3.1 前端数据流

1. **页面加载**：前端页面加载时，通过 API 请求获取个人资料数据
2. **用户交互**：用户与页面交互，如点击按钮、填写表单等
3. **状态更新**：前端状态根据用户交互进行更新
4. **数据提交**：管理员在控制面板中编辑内容并提交，通过 API 发送到后端

#### 3.3.2 后端数据流

1. **请求接收**：后端接收前端发送的 HTTP 请求
2. **认证验证**：对于需要权限的请求，验证用户身份和权限
3. **业务处理**：根据请求类型执行相应的业务逻辑
4. **数据操作**：读取或写入数据文件
5. **响应返回**：将处理结果返回给前端

### 3.4 安全设计

1. **密码加密**：管理员密码使用 bcrypt 进行加密存储
2. **JWT 认证**：使用 JWT 进行无状态认证，减少服务器存储压力
3. **权限控制**：基于角色的权限控制，确保只有管理员可以编辑内容
4. **输入验证**：对所有用户输入进行验证，防止恶意输入
5. **错误处理**：统一的错误处理机制，避免暴露敏感信息

在 `backend/.env` 文件中配置以下环境变量：

| 变量名 | 默认值 | 描述 |
|--------|--------|------|
| JWT_SECRET | bilink_super_secret_key_change_in_production | JWT 密钥，用于生成和验证令牌 |

## 5. 安装部署步骤

### 5.1 开发环境部署

#### 5.1.1 克隆项目

```bash
# 克隆项目到本地
git clone https://github.com/your-username/bilink.git

# 进入项目目录
cd bilink
```

#### 5.1.2 安装后端依赖

```bash
# 进入后端目录
cd backend

# 安装依赖
npm install

# 复制环境变量文件（如果不存在）
if not exist .env (copy .env.example .env)
```

#### 5.1.3 安装前端依赖

```bash
# 返回项目根目录
cd ..

# 进入前端目录
cd frontend

# 安装依赖
npm install
```

#### 5.1.4 启动后端服务器

```bash
# 进入后端目录
cd ..\backend

# 启动后端服务器（开发模式）
npm run dev

# 或启动后端服务器（生产模式）
npm start
```

#### 5.1.5 启动前端服务器

```bash
# 打开新的终端窗口
# 进入前端目录
cd frontend

# 启动前端开发服务器
npm run dev
```

#### 5.1.6 访问应用

- 前端应用：http://localhost:3000/（如果 3000 端口被占用，会自动使用其他端口）
- 后端 API：http://localhost:5000/
- 健康检查：http://localhost:5000/api/health

### 5.2 生产环境部署

#### 5.2.1 构建前端应用

```bash
# 进入前端目录
cd frontend

# 构建生产版本
npm run build

# 构建产物会生成在 dist 目录中
```

#### 5.2.2 部署建议

1. **前端部署**：
   - 将 `frontend/dist` 目录部署到静态网站托管服务（如 Netlify、Vercel、GitHub Pages 等）
   - 或部署到 Nginx、Apache 等 Web 服务器

2. **后端部署**：
   - 将后端代码部署到 Node.js 服务器（如 AWS EC2、DigitalOcean Droplet、Heroku 等）
   - 确保设置正确的环境变量
   - 考虑使用 PM2 等进程管理器来管理 Node.js 应用

3. **环境变量配置**：
   - 在生产环境中，务必修改 `JWT_SECRET` 为强密码
   - 根据部署环境调整 `PORT` 和其他配置

4. **CORS 配置**：
   - 在生产环境中，建议配置具体的前端域名，而不是使用通配符 `*`
   - 修改 `server.js` 中的 CORS 配置：
     ```javascript
     app.use(cors({
       origin: 'https://your-frontend-domain.com'
     }))
     ```

### 5.3 常见部署问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 端口被占用 | 其他应用正在使用相同端口 | 停止占用端口的应用，或修改配置文件中的端口号 |
| 环境变量未配置 | 缺少必要的环境变量 | 检查并创建 .env 文件，确保所有必要的环境变量都已设置 |
| 依赖安装失败 | 网络问题或 Node.js 版本不兼容 | 检查网络连接，确保使用兼容的 Node.js 版本 |
| CORS 错误 | 跨域资源共享配置问题 | 检查后端 CORS 配置，确保允许前端域名 |


## 6. 使用指南

### 6.1 普通访客使用指南

#### 6.1.1 浏览主页内容

1. **访问主页**：打开浏览器，输入前端应用地址（如 http://localhost:3001/）
2. **浏览各个 section**：
   - **Hero 区域**：查看用户头像、姓名、头衔和简短介绍
   - **关于我**：了解用户的详细背景和经历
   - **技能专长**：查看用户的专业技能和熟练度
   - **项目作品**：浏览用户的项目案例
   - **联系方式**：获取用户的联系信息

#### 6.1.2 访问社交链接

1. 在 Hero 区域下方，点击社交链接按钮
2. 系统会在新标签页中打开对应的社交平台页面

#### 6.1.3 联系用户

1. 在联系方式 section 中，点击邮箱地址
2. 系统会打开默认邮件客户端，自动填写收件人地址

### 6.2 管理员使用指南

#### 6.2.1 登录系统

1. **触发登录按钮**：在主页上点击任意位置 3 次，右上角会显示登录按钮
2. **点击登录按钮**：进入登录页面
3. **输入凭据**：
   - 用户名：默认 `admin`
   - 密码：默认 `admin123`
4. **点击登录**：成功登录后，会自动跳转到管理员控制面板

#### 6.2.2 管理个人信息

1. 在管理员控制面板中，点击 "基本信息" 标签
2. 编辑以下信息：
   - 姓名：用户的姓名
   - 头衔：用户的职业头衔
   - 头像 URL：用户头像的链接
   - 简短介绍：一句话介绍
   - 关于我：详细的个人介绍
3. 点击页面底部的 "保存更改" 按钮
4. 系统会显示保存成功的提示

#### 6.2.3 管理技能

1. 在管理员控制面板中，点击 "技能专长" 标签
2. **添加技能**：
   - 点击 "添加技能" 按钮
   - 填写技能名称、图标和熟练度
3. **编辑技能**：
   - 修改技能的名称、图标或熟练度
4. **删除技能**：
   - 点击技能项右侧的删除按钮
5. 点击页面底部的 "保存更改" 按钮

#### 6.2.4 管理项目

1. 在管理员控制面板中，点击 "项目作品" 标签
2. **添加项目**：
   - 点击 "添加项目" 按钮
   - 填写项目名称、描述、链接、图片 URL 和标签（逗号分隔）
3. **编辑项目**：
   - 修改项目的各项信息
4. **删除项目**：
   - 点击项目项右侧的删除按钮
5. 点击页面底部的 "保存更改" 按钮

#### 6.2.5 管理社交链接

1. 在管理员控制面板中，点击 "社交链接" 标签
2. **添加链接**：
   - 点击 "添加链接" 按钮
   - 选择链接类型、填写显示名称和链接地址
3. **编辑链接**：
   - 修改链接的类型、显示名称或地址
4. **删除链接**：
   - 点击链接项右侧的删除按钮
5. 点击页面底部的 "保存更改" 按钮

#### 6.2.6 管理联系方式

1. 在管理员控制面板中，点击 "联系方式" 标签
2. 编辑以下信息：
   - 电子邮箱：用户的邮箱地址
   - 所在地：用户的地理位置
3. 点击页面底部的 "保存更改" 按钮

#### 6.2.7 退出登录

1. 在管理员控制面板的右上角，点击 "退出登录" 按钮
2. 系统会清除登录状态，并跳转到主页

#### 6.2.8 账户管理

1. 在管理员控制面板中，点击 "账户管理" 标签

2. **管理账户信息**：
   - 在 "账户信息" 部分修改用户名
   - 角色字段为只读（系统目前只支持管理员角色）
   - 点击 "更新账户信息" 按钮提交修改

3. **重置密码**：
   - 在 "密码重置" 部分输入当前密码
   - 输入新密码（密码要求：长度不少于8位，包含大小写字母及特殊符号）
   - 再次输入新密码进行确认
   - 点击 "更新密码" 按钮提交修改

4. **密码要求**：
   - 长度不少于8位
   - 包含大写字母
   - 包含小写字母
   - 包含特殊符号

5. **操作状态**：
   - 操作过程中会显示加载动画
   - 操作成功时会显示绿色的成功提示
   - 操作失败时会显示红色的失败提示

### 6.3 使用注意事项

1. **安全注意事项**：
   - 首次登录后，请修改默认密码
   - 不要在公共场合使用管理员账户
   - 定期更新密码

2. **数据管理注意事项**：
   - 每次修改内容后，务必点击 "保存更改" 按钮
   - 上传的图片建议使用 CDN 或其他可靠的图片托管服务
   - 技能熟练度范围为 0-100

3. **性能优化注意事项**：
   - 避免添加过多的项目或技能，以免影响页面加载速度
   - 图片文件建议压缩后使用
   - 社交链接数量建议控制在 5-8 个之间


## 7. 常见问题解答

### 7.1 前端问题

#### 7.1.1 页面加载失败，显示空白页面

**问题描述**：访问前端应用时，页面显示空白，控制台有错误信息。

**可能原因**：
- 前端依赖未正确安装
- 后端服务器未启动
- API 请求路径错误

**解决方案**：
1. 检查前端依赖是否已正确安装：
   ```bash
   cd frontend
   npm install
   ```
2. 确保后端服务器已启动：
   ```bash
   cd backend
   npm run dev
   ```
3. 检查浏览器控制台的错误信息，确认 API 请求路径是否正确

#### 7.1.2 液态玻璃效果不显示

**问题描述**：页面元素没有显示液态玻璃效果，看起来只是普通的半透明效果。

**可能原因**：
- 浏览器不支持 `backdrop-filter` 属性
- CSS 样式未正确加载

**解决方案**：
1. 使用支持 `backdrop-filter` 属性的现代浏览器（如 Chrome、Firefox、Safari 10+、Edge 17+）
2. 检查浏览器开发者工具中的样式面板，确认 `glass.css` 文件是否已正确加载

#### 7.1.3 登录按钮不显示

**问题描述**：在主页上点击多次后，登录按钮仍然不显示。

**可能原因**：
- 点击次数不足
- 浏览器控制台有 JavaScript 错误

**解决方案**：
1. 确保在主页上点击了至少 3 次
2. 检查浏览器控制台是否有 JavaScript 错误
3. 尝试刷新页面后重新点击

### 7.2 后端问题

#### 7.2.1 API 请求返回 404 错误

**问题描述**：前端发送 API 请求时，返回 404 Not Found 错误。

**可能原因**：
- 后端服务器未启动
- API 路由路径错误
- 后端代码有语法错误

**解决方案**：
1. 确保后端服务器已启动：
   ```bash
   cd backend
   npm run dev
   ```
2. 检查 API 请求路径是否正确，应该以 `/api/` 开头
3. 检查后端服务器的日志，确认是否有语法错误

#### 7.2.2 登录失败，返回 "用户名或密码错误"

**问题描述**：输入正确的用户名和密码后，登录仍然失败。

**可能原因**：
- 用户名或密码错误
- 密码加密过程出错
- 数据文件未正确初始化

**解决方案**：
1. 确认使用的是正确的默认凭据：
   - 用户名：`admin`
   - 密码：`admin123`
2. 检查后端服务器的日志，确认数据文件是否已正确初始化
3. 如果数据文件损坏，可以删除 `backend/data/data.json` 文件，然后重启后端服务器，系统会重新创建默认数据

#### 7.2.3 保存个人资料失败

**问题描述**：在管理员控制面板中修改个人资料后，点击保存按钮，显示保存失败。

**可能原因**：
- 后端服务器未启动
- 认证令牌过期
- 数据文件权限不足

**解决方案**：
1. 确保后端服务器已启动
2. 尝试重新登录，获取新的认证令牌
3. 检查 `backend/data/data.json` 文件的权限，确保有写入权限

### 7.3 部署问题

#### 7.3.1 端口被占用

**问题描述**：启动服务器时，显示 "address already in use" 错误。

**可能原因**：
- 其他应用正在使用相同的端口

**解决方案**：
1. 停止占用端口的应用
2. 或修改配置文件中的端口号：
   - 后端端口：修改 `backend/.env` 文件中的 `PORT` 变量
   - 前端端口：修改 `frontend/vite.config.js` 文件中的端口配置

#### 7.3.2 依赖安装失败

**问题描述**：运行 `npm install` 时，依赖安装失败。

**可能原因**：
- 网络问题
- Node.js 版本不兼容
- npm 缓存问题

**解决方案**：
1. 检查网络连接是否正常
2. 确保使用的是兼容的 Node.js 版本（^24.11.1 或更高）
3. 尝试清除 npm 缓存后重新安装：
   ```bash
   npm cache clean --force
   npm install
   ```

#### 7.3.3 CORS 错误

**问题描述**：前端应用无法访问后端 API，浏览器控制台显示 CORS 错误。

**可能原因**：
- 后端 CORS 配置错误
- 前端请求路径错误

**解决方案**：
1. 确保后端服务器已启动，并且 `cors` 中间件已正确配置
2. 检查前端 API 请求的路径是否正确
3. 在生产环境中，确保 CORS 配置中包含了正确的前端域名

### 7.4 安全问题

#### 7.4.1 忘记管理员密码

**问题描述**：无法登录管理员账户，因为忘记了密码。

**解决方案**：
1. 删除 `backend/data/data.json` 文件
2. 重启后端服务器，系统会重新创建默认数据，包括默认密码 `admin123`
3. 登录后，及时修改默认密码

#### 7.4.2 如何修改管理员密码

**问题描述**：需要修改默认的管理员密码，以提高安全性。

**解决方案**：
1. 停止后端服务器
2. 删除 `backend/data/data.json` 文件
3. 修改 `backend/.env` 文件中的 `ADMIN_PASSWORD` 变量
4. 重启后端服务器，系统会使用新的密码创建管理员账户

### 7.5 其他问题

#### 7.5.1 页面加载速度慢

**问题描述**：前端页面加载速度较慢，特别是在首次访问时。

**可能原因**：
- 图片资源过大
- 依赖包过多
- 网络连接较慢

**解决方案**：
1. 压缩图片资源，使用适当的图片格式
2. 减少不必要的依赖包
3. 考虑使用 CDN 加速静态资源加载

#### 7.5.2 如何备份数据

**问题描述**：需要备份系统中的数据，以防数据丢失。

**解决方案**：
1. 复制 `backend/data/data.json` 文件到安全的位置
2. 定期进行备份，特别是在修改重要数据之前

#### 7.5.3 如何迁移数据到新服务器

**问题描述**：需要将系统从一个服务器迁移到另一个服务器，同时保留所有数据。

**解决方案**：
1. 在原服务器上，复制 `backend/data/data.json` 文件
2. 在新服务器上，安装并配置系统
3. 将复制的数据文件替换新服务器上的 `backend/data/data.json` 文件
4. 重启后端服务器


## 8. 代码注释规范

### 8.1 注释的重要性

代码注释是提高代码可读性和可维护性的重要手段，它可以：

- 帮助开发者理解代码的功能和实现思路
- 减少新开发者的学习成本
- 为代码审查提供参考
- 防止代码被误解或误用
- 记录代码的变更历史和决策依据

### 8.2 注释规范

#### 8.2.1 文件头部注释

每个文件都应该包含文件头部注释，说明文件的基本信息：

```javascript
/**
 * Bilink 个人交互主页系统 - [模块名称]
 * 
 * [文件功能描述]
 * 
 * @file [文件名]
 * @author Bilink Team
 * @version 1.0.0
 * @since 2026-01-13
 */
```

#### 8.2.2 类/函数注释

每个类和函数都应该包含详细的注释，说明其功能、参数和返回值：

```javascript
/**
 * [函数功能描述]
 * 
 * @param {[类型]} [参数名] - [参数描述]
 * @returns {[类型]} [返回值描述]
 * @throws {[异常类型]} [异常描述]
 * @example
 * [使用示例]
 */
function exampleFunction(parameter) {
  // 函数实现
}
```

#### 8.2.3 行内注释

对于复杂的代码逻辑，应该添加行内注释，说明代码的实现思路：

```javascript
// [注释内容]
const result = complexOperation(); // [行内注释]
```

#### 8.2.4 注释的格式

- 使用 `/** */` 进行多行注释
- 使用 `//` 进行单行注释
- 注释应该与代码保持一致的缩进
- 注释应该使用简洁明了的语言
- 注释应该准确反映代码的功能

#### 8.2.5 注释的内容

注释应该包含以下内容：

- 代码的功能和目的
- 实现思路和算法说明
- 关键变量和常量的含义
- 潜在的问题和注意事项
- 性能优化的考虑
- 与其他代码的依赖关系

### 8.3 注释示例

#### 8.3.1 前端代码注释示例

```javascript
/**
 * 认证提供者组件
 * 
 * 为应用提供全局认证状态管理
 * 
 * @param {Object} props - 组件属性
 * @param {React.ReactNode} props.children - 子组件
 * @returns {JSX.Element} 认证提供者组件
 */
export const AuthProvider = ({ children }) => {
  // 认证状态
  const [user, setUser] = useState(null) // 当前登录用户信息
  const [loading, setLoading] = useState(true) // 加载状态

  /**
   * 初始化认证状态
   * 
   * 组件挂载时，检查本地存储中是否存在令牌
   * 如果存在令牌，则验证令牌有效性
   */
  useEffect(() => {
    const token = localStorage.getItem('token')
    if (token) {
      verifyToken(token)
    } else {
      setLoading(false)
    }
  }, [])

  // 其他代码...
}
```

#### 8.3.2 后端代码注释示例

```javascript
/**
 * 从文件加载数据
 * 
 * @returns {Object|null} 加载的数据对象，如果加载失败则返回 null
 */
export function loadData() {
  try {
    if (fs.existsSync(DATA_FILE)) {
      const raw = fs.readFileSync(DATA_FILE, 'utf8')
      return JSON.parse(raw)
    }
  } catch (err) {
    console.error('Error loading data:', err)
  }
  return null
}
```

### 8.4 注释的最佳实践

- **保持注释更新**：当代码变更时，确保注释也相应更新
- **避免冗余注释**：不要注释显而易见的代码
- **使用一致的风格**：在整个项目中使用统一的注释风格
- **注重质量而非数量**：注释应该有实际价值，而不是为了注释而注释
- **使用工具辅助**：考虑使用 ESLint 等工具来检查注释的规范性
